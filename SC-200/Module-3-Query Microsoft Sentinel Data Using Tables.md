# Module 3 — Query Microsoft Sentinel Data Using Tables

## Microsoft SC-200: Security Operations Analyst
---

# Module Overview

Microsoft Sentinel stores security telemetry inside structured tables within an Azure Log Analytics Workspace.

Every log, alert, event, authentication record, endpoint activity, and security finding eventually becomes searchable data inside a table.

Understanding Sentinel tables is critical because nearly every security activity depends on them:

* Threat Hunting
* Incident Investigation
* Analytics Rules
* Detection Engineering
* Security Monitoring
* Forensics
* KQL Queries

This module introduces the most common tables used by SOC analysts and security engineers.

---

# 1. Why Tables Matter

## The Foundation of Security Operations

Modern organizations generate telemetry from:

* Endpoints
* Identity Systems
* Servers
* Firewalls
* Cloud Platforms
* Security Products

Without structured storage, investigations would become extremely difficult.

Tables provide:

✅ Organization

✅ Searchability

✅ Scalability

✅ Investigation Efficiency

✅ Threat Hunting Capability

## Purpose

Stores:

> Windows Security Event Logs

Collected from:

* Windows Servers
* Domain Controllers
* Windows Endpoints

---

# Common Use Cases

* Successful Login Analysis
* Privilege Escalation
* Account Lockouts

---

# Important Event IDs

| Event ID | Meaning                 |
| -------- | ----------------------- |
| 4624     | Successful Login        |
| 4625     | Failed Login            |
| 4720     | User Created            |
| 4728     | Added to Security Group |

# MOST IMPORTANT EXAM FACTS

✅ SecurityEvent = Windows Logs

✅ SignInLogs = Entra ID Authentication

✅ AuditLogs = Administrative Changes

✅ SecurityAlert ≠ SecurityIncident

✅ Events → Alerts → Incidents

✅ DeviceEvents = Endpoint Telemetry

---

# Analyst Takeaways

* Microsoft Sentinel organizes telemetry into structured tables.
* Table selection is often the first step of an investigation.
* Identity investigations rely heavily on SignInLogs and AuditLogs.
* Endpoint investigations commonly use DeviceEvents.
* SecurityAlert and SecurityIncident serve different purposes.
* Understanding table usage is foundational for KQL, hunting, and investigations.
