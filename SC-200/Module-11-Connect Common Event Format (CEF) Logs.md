# Module 11 — Connect Common Event Format (CEF) Logs

## Microsoft SC-200: Security Operations Analyst
---

# Module Overview

Organizations rarely use only Microsoft security products.

Most enterprise environments contain:

* Firewalls
* IDS/IPS Platforms
* VPN Appliances
* Routers
* Switches
* Security Gateways
* Third-Party Security Solutions

Microsoft Sentinel uses the Common Event Format (CEF) connector to collect and normalize security logs from these products.

This module introduces:

* Common Event Format (CEF)
* Linux Log Forwarders
* Azure Monitor Agent (AMA)
* Third-Party Log Collection Architecture
* CEF vs Syslog

Understanding this architecture is critical for both the SC-200 exam and real-world SIEM deployments.

# 1. What is CEF?

## Definition

CEF stands for:

> Common Event Format

---

# Purpose

Provide a standardized format for security events generated by different vendors.

---

# Why It Exists

Different vendors use different:

* Field Names
* Event Structures
* Log Formats

CEF normalizes these differences.

---

# Benefits

✅ Consistent Data Structure

✅ Easier Parsing

✅ Better Correlation

✅ Improved Visibility

✅ Multi-Vendor Compatibility

---

# SOC Perspective

Analysts can investigate security events more efficiently when logs follow a consistent format.

# 7. CEF vs Syslog

One of Microsoft's favorite exam distinctions.

---

# CEF

Purpose:

Defines the structure of security events.

---

# Think

```text id="v5l5r6"
CEF
=
How Logs Look
```

---

# Syslog

Purpose:

Transports log messages between systems.

---

# Think

```text id="j2v5dw"
Syslog
=
How Logs Travel
```

---

# Comparison Table

| CEF                 | Syslog              |
| ------------------- | ------------------- |
| Event Format        | Transport Protocol  |
| Defines Structure   | Moves Data          |
| Standardizes Events | Delivers Events     |
| Data Appearance     | Data Transportation |

# Interview-Level Questions

## Q1. What is CEF?

A standardized security event format used by multiple vendors.

---

## Q2. What is the purpose of a Linux Log Forwarder?

To receive and forward CEF logs to Azure.

---

## Q3. What is Microsoft's preferred monitoring agent?

Azure Monitor Agent (AMA).

---

## Q4. What is the difference between CEF and Syslog?

CEF defines event structure.

Syslog transports logs.

## Q5. Why is CEF important?

It enables standardized log ingestion from third-party products.
